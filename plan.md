1. Файловая Архитектура Оценка: 2/10

Что хорошо: структура простая и понятная для маленького статического сайта.

Проблемы:

index.html:1-386 — весь лендинг в одном HTML-файле.
css/style.css:1-2050 — god-file: все стили, токены, секции, анимации и responsive в одном файле.
js/main.js:1-303 — весь интерактив в одном файле.
assets/ — пустая директория.
Нет ui / sections / layout / lib / i18n.
Рекомендация: разбить проект минимум на sections, components, styles, data, scripts. Если цель production/senior — переносить на Next.js или хотя бы Vite с модульной структурой.

Проблемы:

main.js:1-303 — несколько независимых ответственностей в одном файле.
index.html:86-332 — данные, структура и представление смешаны.
index.html:156-185 — карточки услуг захардкожены, расширение требует правки HTML.
index.html:294-330 — форма выглядит как форма, но фактически CTA-ссылка.
Рекомендация: вынести данные портфолио, услуг, этапов и CTA в конфиги. Логику разделить на отдельные модули.

3. DRY Оценка: 2/10

Проблемы:

css/style.css:222-270 + main.js:1-65 — кастомный курсор добавляет сложность и риски accessibility.
css/style.css:433-487 — hero visual layer довольно тяжёлый для простого лендинга.
main.js:1-303 — файл уже сложнее, чем должен быть для поддержки.
Рекомендация: оставить визуальные эффекты, но изолировать их и дать graceful fallback.

5. Чистота Кода Оценка: 4/10

Что хорошо: HTML аккуратно отформатирован, классы в целом консистентные по BEM-подходу.

Проблемы:

main.js:1,68,82,119,150,176,198,276,291 — много комментариев-заголовков.
css/style.css:60 — глобально скрывается системный курсор через * { cursor: none; }.

index.html:105 и index.html:114 — карточка визуально ведёт на atlanttech.pro, а CTA внутри на ocenka-biznesa-spb.ru; это выглядит как баг данных.
